아하 오케이!
네가 표로 정리한 알고리즘들(1.3 ~ 7.2)에 “대응되는 백준 문제 번호”만 깔끔하게 알려달라는 거지?
그러면 딱 그 알고리즘과 가장 직접적으로 연결되는 BOJ 문제만 골라서 정확히 매칭해줄게.

⸻

🟨 1.3 ~ 7.2 알고리즘별 백준 문제 번호 매핑

아래 문제들은 그 알고리즘을 그대로 쓰거나, 가장 가까운 개념을 사용하는 대표 문제들이야.

⸻

✅ 1.3 교환정렬 (Exchange Sort)

→ “모든 쌍 비교, 교환” = 기초 정렬
	•	BOJ 2750 — 수 정렬하기 (B2)
	•	BOJ 2751 — 수 정렬하기 2 (S5, 더 큰 입력 크기)
(교환정렬 그대로는 아니지만, 정렬 기본기 문제로 대응)

⸻

✅ 4.5 허프만 코드 생성 (Huffman Coding)

(Huffman Tree = 최소 두 개 꺼내서 합치기 = Min-Heap)
	•	BOJ 1715 — 카드 정렬하기 (G4)
	•	BOJ 15903 — 카드 합체 놀이 (S1)
→ 둘 다 Huffman 알고리즘 원리 그대로

⸻

✅ 5.1 n-Queens (Backtracking)
	•	BOJ 9663 — N-Queen (G4)
→ 완벽히 동일한 문제

⸻

✅ 5.4 부분집합의 합 (Sum of Subsets)
	•	BOJ 1182 — 부분수열의 합 (S2)
→ 부분집합 전체 탐색 + 백트래킹

⸻

✅ 5.5 m-색칠하기 (Graph Coloring)

BOJ에는 “m-coloring”을 직접 요구하는 문제는 거의 없음.
대신 그래프 색칠의 본질: 인접 노드는 다른 색과 동일한 문제는 아래.
	•	BOJ 1707 — 이분 그래프 (G4)
→ 2-coloring 문제
→ m-coloring의 가장 대표적인 특수 사례

⸻

✅ 5.7 0–1 배낭 백트래킹

BOJ에서 knapsack은 대부분 DP로 출제.
백트래킹 구조 그대로 쓰는 문제는 없음.
가장 가까운 대응은:
	•	BOJ 12865 — 평범한 배낭 (G5, DP 해결)
(이론은 동일 문제 / 구현 방식만 다름)

⸻

✅ 6.1 0-1 배낭 Branch-and-Bound (BFS)

Branch-and-Bound 방식은 BOJ에서 직접 요구 거의 없음.
근본 문제는 역시 0–1 knapsack:
	•	BOJ 12865 — 평범한 배낭 (G5)
(BFS/BB로 풀 수는 있지만 실전에서는 DP 사용)

⸻

✅ 6.2 0-1 배낭 분기한정 Best-First Search
	•	BOJ 12865 — 평범한 배낭 (G5)

(BOJ에서는 Best-first knapsack이 출제되지 않음)

⸻

✅ 6.3 분기한정 TSP (Traveling Salesman Problem)
	•	BOJ 2098 — 외판원 순회 (G1)
→ TSP의 정석 문제 (보통 DP+Bitmask로 풀이)

⸻

✅ 7.1 삽입정렬 (Insertion Sort)

BOJ에 직접 “삽입정렬을 구현하라”는 문제는 몇 개 있음.
	•	BOJ 11399 — ATM (S3)
→ 최소값을 앞으로 가져오는 사고가 insertion sort 사고와 동일
	•	BOJ 24051 — 알고리즘 수업 - 삽입 정렬 1 (S4)
→ 실제 삽입정렬 코드 기반 문제
	•	BOJ 24052 — 알고리즘 수업 - 삽입 정렬 2 (S4)

⸻

✅ 7.2 선택정렬 (Selection Sort)

BOJ에서 선택정렬 구현 문제 그대로 나옴.
	•	BOJ 23882 — 알고리즘 수업 - 선택 정렬 1 (S4)
	•	BOJ 23883 — 알고리즘 수업 - 선택 정렬 2 (S4)
